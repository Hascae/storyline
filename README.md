# 物語時鐘 · Monogatari Clock

> 把一天讀成一個故事的時鐘。

一款現代、簡約、可靠的 Android 時鐘應用。以 Flutter 打造，介面與美術資源全部原創手工設計 —— 沒有卡片、沒有毛玻璃，只有紙、墨、朱與留白。

## 設計語言：一日四章

「物語」即故事。物語時鐘把一天分為四個章節，整個介面的紙色、墨色與朱色隨時辰自然流轉：

| 章節 | 時段 | 基調 |
| --- | --- | --- |
| 晨之章 | 05:00 – 09:00 | 微暖的生成り紙色，晨光初染 |
| 晝之章 | 09:00 – 17:00 | 明亮的紙白與藍墨，日色正好 |
| 暮之章 | 17:00 – 20:00 | 帶紫的暮色與茜紅，一章將盡 |
| 夜之章 | 20:00 – 05:00 | 沉靜的墨色與朱色，故事低語 |

配色取自日本傳統色（生成り、墨、朱、藍、茜），輔以髮絲線、縱書章節籤與一方「物」字落款印。所有圖標一筆一畫以路徑手繪，錶盤、開關、滾輪皆為自繪部件，不使用任何系統圖標素材；鬧鐘與計時提示音為原創合成的風鈴鐘聲。

## 功能

- **時鐘** —— 自繪指針錶盤 + 大字數位時刻，隨章節流轉的問候詩句
- **世界時鐘** —— 50+ 城市，中文／拼音／英文搜尋，時差與昨天／今天／明天、晝夜一目了然
- **鬧鐘** —— 單次與週期（每天／工作日／週末／自選星期）、標籤、振動開關、稍後再響間隔可調（5／10／15 分鐘）、**跳過下一次響鈴**、距下次響鈴倒數
- **響鈴頁** —— 鎖屏全屏喚起，大目標按鈕不怕按錯；通知上可直接「稍後再響／停止」
- **計時器** —— **多計時器並行**、常用預設、+1 分鐘、暫停續行，殺進程後由牆鐘時間精準復原
- **碼錶** —— 百分秒逐幀顯示、計次與最快／最慢標記，後台與重啟均不丟計時

## 可靠性：到點必響

鬧鐘應用的生命線是通知。物語時鐘在代碼層面做足：

- `AlarmManager` **alarmClock 級精確排程**（`AndroidScheduleMode.alarmClock`），Doze 省電模式下依然準點
- 聲明 `USE_EXACT_ALARM`（Android 13+ 鬧鐘應用由系統自動授予）與 `SCHEDULE_EXACT_ALARM`（Android 12/12L），不需用戶手動開關
- `RECEIVE_BOOT_COMPLETED`：重啟、應用更新後自動把所有鬧鐘重新掛回系統
- 通知走**鬧鐘專用音訊通道**（`AudioAttributesUsage.alarm`），媒體音量靜音也照響；鈴聲循環直至處理，8 分鐘無人理會自動靜音
- **預埋回響**：主鈴排入系統的同時，「稍後再響間隔」與其兩倍處各預排一發回響。稍後再響到點必響、睡過頭自動再響，皆不依賴任何後台執行；按「停止」即撤
- 全屏意圖（Full-Screen Intent）在鎖屏直接亮出響鈴頁
- 週期鬧鐘按星期展開為系統級每週重複排程，即使應用長期不被打開也持續有效
- 每次啟動應用皆對排程做一次對賬重掛，防個別 ROM 清理

授權皆由系統彈窗完成（Android 13+ 的通知權限等），介面內不設任何授權按鈕。

## 語言

簡體中文與繁體中文，**完全跟隨系統語言自動切換**，無語言設定項：

- 系統為繁體（zh-Hant／台灣／香港／澳門）→ 繁體中文
- 其他 → 簡體中文

應用名稱同樣隨系統語言顯示為「物語時鐘」或「物语时钟」。

## 構建

### GitHub Actions（推薦）

倉庫附帶手動觸發的正式構建流程：

1. GitHub → **Actions** → **構建發佈版 APK** → **Run workflow**
2. 流程依序執行：依賴解析 → 本地化生成 → 靜態分析 → 單元測試 → 發佈版構建
3. 完成後在 Artifacts 下載 `monogatari-clock-release-unsigned`

產出為**未簽名**的發佈版 APK，並在流程中驗證確實無簽名。安裝前請自行簽名：

```bash
apksigner sign --ks your.keystore --out app-release.apk app-release-unsigned.apk
```

### 本地

```bash
flutter pub get
flutter test
flutter build apk --release
# 產物：build/app/outputs/flutter-apk/app-release.apk（未簽名）
```
